## 六、附录：技术术语速查卡

### 🧩 AI/Agent领域专业词汇

| 英文术语 | 中文含义 | 使用场景 |
|---------|---------|---------|
| **Agent Orchestration** | 代理编排 | 多Agent协同工作调度 |
| **Task Decomposition** | 任务分解 | 将复杂任务拆解为子任务 |
| **Role-based Agents** | 基于角色的代理 | 模拟不同职能的智能体 |
| **Hallucination Mitigation** | 幻觉缓解 | 减少LLM生成虚假内容 |
| **RAG Pipeline** | 检索增强生成管道 | 结合外部知识的问答流程 |
| **Context Window** | 上下文窗口 | LLM一次能处理的文本长度 |
| **Prompt Engineering** | 提示工程 | 优化输入指令的艺术 |
| **Inference Latency** | 推理延迟 | API响应时间指标 |
| **Token Usage Cost** | Token使用成本 | 大模型调用费用计量单位 |
| **Production Readiness** | 生产就绪 | 系统可上线运行的状态 |

### 📐 常用度量指标

```
✅ 项目影响力:
• GitHub Stars/Forks/Rate

• 文档页面月阅读量
• 社区Issue响应速度(Mean Time To Response)

✅ 工程质量:
• 代码覆盖率(Code Coverage > 85%)
• CI/CD通过率(> 95%)
• 平均故障修复时间(MTTR < 4小时)
• API可用性SLA (> 99.5%)
```
